반복문

반복문의 정의, while문, for문, 반복문의 제어(break,continue), 반복문 중첩, 문제로 진행할 예정입니다.

1. 반복문이란?

  • 반복문이란 프로그램 내에서 똑같은 명령을 일정 횟수만큼 반복하여 수행하도록 제어하는 실행문입니다.
  • 프로그램이 처리하는 대부분의 코드는 반복적인 형태가 많으므로, 가장 많이 사용되는 실행문 중 하나입니다.
  • 반복문을 쓰는 이유 : 여러개의 같은 코드를 반복하고 싶을 때, 자료형에 담긴 자료들을 하나씩 꺼내고 싶을 때 사용
  • while문/for문(~하는 동안)

2. while문이란?

  • while 문은 특정 조건을 만족할 때까지 계속해서 주어진 실행문을 반복 실행합니다.

  • while문의 문법은 다음과 같습니다.

    while( 표현식 ) {
    실행문
    }

    표현식에는 boolean 값이여야 합니다. 즉 true 또는 false로 나와야합니다.

    ex.1.)
    while(true){
    console.log("방장님쵝오")
    }
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    ex.2.)
    while(false){
    console.log("으하하하")
    }
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    콘솔창에서 보고나니깐 어떤 느낌인지 아시겠나요? 모르면 연락해요 알려드릴게요^^

  • 첫번째 예시를 콘솔창에 돌렸다가 페이지 마비오죠???? 저도 테스트하다가 아차 했답니다..

    - 이렇게 계속해서 끝이 안보이는 로직을 무한루프에 빠졌다고 합니다. 무한루프에 안걸리고싶으면 어떻게 해야할까요?

    ex.3.)

    이런식으로 표현식 결과가 false가 나오도록 표현식을 변경하는 실행문이 만들어져야겠죠?

3. do-while문이란?

  • while 문은 루프에 진입하기 전에 먼저 표현식부터 검사합니다. 하지만 do-while 문은 먼저 루프를 한 번 실행한 후에 표현식을 검사합니다.
  • 즉, do / while 문은 표현식의 결과와 상관없이 무조건 한 번은 루프를 실행합니다.

  • do-while문의 문법은 다음과 같습니다.

    do {
    실행문
    } while(표현식);

    while문에서 감을 잡으셨으면 do-while은 어머?! 내가 왜 알고 있지?! 할거에요.. 그래줘요.. 제발..

    ex.1.)
    let i = 1, j = 1
    while(i > 3){
    console.log("i : " + (i++));
    }
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    ex.2.) // i와 j 는 위에서 선언해 준걸로 사용할게요
    do{
    console.log("j : " + (j++));
    }while(j > 3);
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    while 문은 표현식에서 false나와서 반복이 안되고 do-while은 우선 실행문을 돌고나서 표현식을 따지니 한 번 도네요!!

4. for문이란?

  • for 문은 while 문과는 달리 자체적으로 초기식, 표현식, 증감식을 모두 포함하고 있는 반복문입니다.
  • while 문보다는 좀 더 간결하게 반복문을 표현할 수 있습니다.

  • for문의 문법은 다음과 같습니다.

    for(초기식; 표현식; 증감식){
    실행문
    }

    while문 do-while문을 사용했다면 for문은 금방 이해하실거에요. 예시를 들어보도록 하겠습니다.

    ex.1.)
    for(var i = 0; i < 10; i++){
    console.log("i = " + i)
    } => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    ex.2.)
    var sum = 0;
    for(var i = 0; i < 10; i++){
    sum = sum + i
    i++
    } console.log("sum = " + sum)
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

5. break와 continue란?

  • break는 반복작업을 중간에 중단시키고 싶을 때 사용합니다.
  • continue는 실행을 즉시 중단하면서 반복을 지속하게 할 때 사용합니다.

  • break는 다음과 같습니다.

    for(var i=0; i < 10; i++){
    if(i == 5){
    break;
    }
    console.log(i);
    }
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    이해가 가나요? break를 사용했을 때 반복문이 끝나죠?

    continue는 다음과 같습니다.

    for(var i=0; i < 10; i++){
    if(i == 5){
    continue;
    }
    console.log(i);
    }
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    i의 값이 5가 되었을 때 실행이 중단 됐기 때문에 콘솔창에 찍힌 값처럼 나옵니다. contine이후 구문이 실행이 안됩니다. 하지만 반복문은 중단되지 않았기 때문에 나머지 결과가 출력된 것!

.6 for/in문이란?

  • for/in 문은 일반적인 for문과는 전혀 다른 형태의 반복문입니다.(열거형)
  • for/in 문은 객체의 key 값에 접근할 수 있습니다.
  • 주로 객체에 사용합니다. 즉, 객체 자료형에 자료들을 하나씩 꺼내고 싶을 때 사용합니다.

  • for/in문의 문법은 다음과 같습니다.

    for(변수명 in 객체식){
    실행문
    }

    예시를 보는게 이해하기 좋을 것 같습니다.

    예시1)
    var obj = {
    x : 1,
    y : true,
    z : 'sumi'
    };
    for(var k in obj) {
    console.log(k + " : " + obj[k]);
    }
    => 어떻게 나올까요? 답은 콘솔창에 ctrl+c/ctrl+v를 해보시길 바랍니다.

    예제1의 객체 obj에는 세 개의 프로퍼티가 있으며 for/in 문에서 프로퍼티명인 x,y,z 문자열 변수 k에 대입됩니다.
    프로퍼티값을 표시할 때는 obj[k]와 같이 대괄호 연산을 사용합니다.

    for/in문에서 주의할 사항

    • 1. 프로퍼티를 열거하는 순서
      • 순서가 보장되는 것이 아닙니다. 원래 프로퍼티 간에는 순서가 없는 것으로 순서를 의식하는 것 자체가 잘못입니다.
      • 배열의 경우는 순서를 의식하는 데이터이고요
    • 2. 열거할 수 없는 프로퍼티의 존재

    for/in 문 특징

    • enumerable 한 것들만 출력합니다.
      • 객체자료형을 만들 때 단순히 개발자가 적은 키와 값만 저장되는 것이 아닙니다.
      • 그 외에 숨겨진 속성 3개가 같이 저장됩니다.
      • enumerable 이 true인 값만 출력됩니다.
      • Obejct.getOwnPropertyDescriptor(객체명,'객체키명') //콘솔창에 작성해보기
    • 객체의 prototype도 반복문으로 출력이됩니다. 이것은 생성자를 공부하고 난 뒤 알아가봅시다.

.6 for/of문이란?

  • for/in 문은 for/in 반복문과 유사하지만 배열에 주로 사용합니다.
  • 배열 안에 있는 자료들을 하나씩 꺼내고 싶을 때 사용하는 것 입니다.
  • 배열 뿐만아니라 문자, map, set 등 과 같은 자료형에도 사용이 됩니다.(iterable(symbol.iterator이라는 메서드를 지니고 있는 자료형)인 자료형들에만 적용 가능)
    • 변수명[Symbol.iterator]() 라는 메서드를 출력하면 Array Iterator{} 이라는 이상한 것이 출력됩니다.
    • 반복문 출력을 가능하게 해주는 일종의 함수라고 생각하시면 됩니다. 즉, 저게 있어야 for of 반복문이 가능하다고 보면 됩니다.
