Prototype(유전자)
function 기계(){ this.코드 this.코드 this.코드 }
라고 되어 있으면 생성자 임 해서 인스턴스 생성은
var 변수 = new 기계() 이렇게 생성을 한다
근데 생성 하는 정확한 과정은 new 라는 키워드는 오브젝트나 어레이를 만들때 사용 되는데
var arr =[1,2,3] 이라고 명령어를 만들면 var arr = new Array(1,2,3) 이라고 컴퓨터는 인식을 한다 여기서 Array라는 생성자에 없는 기능들, 메소드 들을
추가 하거나 원하는 값을 찾을 때 prototype을 이용한다 자식에게는 없는 속성을 부모에게서 찾는다 정확한 순서는

1. 생성자에 기능이나 속성이 있는지 찾아보고
2. 없으면 부모에게 가서 찾는다
3. 부모에게도 없으면 상위 노드나 부모의 부모에게 찾는다
4. 찾으면 그 노드 protype으로 속성이 선언 되어 있거나, 아니면 메소드로 되어 있음
좀더 심화 과정으로
Array 나 Object에는 내장 함수들이 포함 되어 있는데 prototype으로 확인이 가능하다



Prototype 이랑 Consturctor 비교

자식들에게 직접 값들을 가지고 싶게 하면 Consturctor로 상속시키면 됨
부모만 가지고 있고 그걸 참조해서 쓰게 하고 싶으면 protytpe으로 하면 됨
보통은 상속 할 수 있는 함수들은 prototype으로 만듬
